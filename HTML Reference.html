Document Structure Tags

<!DOCTYPE>
Defines the document type and version of HTML. It should be the first thing in an HTML document.
Attributes: None (write exactly as <!DOCTYPE html> for HTML5) 
<!DOCTYPE html>
Indicates the document is HTML5
<html>
The root element of an HTML page. Holds all HTML content.
Attributes:
  • lang: Specifies language of the document (e.g., en, fr)
  • dir: Text direction (ltr, rtl, auto)
<html lang="en" dir="ltr">
  ...
</html>
Root of the HTML document representing English language with left-to-right text direction
<head>
Contains meta-information about the document, scripts, styles, and title.
Attributes: None 
<head>
  <title>Page Title</title>
  <meta charset="UTF-8">
  <link rel="stylesheet" href="styles.css">
</head>
Metadata and resource linking goes here (not shown visually)
<body>
Contains the visible content of the HTML document.
Attributes:
  • onload: Script to run when page finished loading
  • Global attributes allowed
<body>
  <h1>Hello World</h1>
  <p>This is page content.</p>
</body>

Hello World

This is page content.

Links and Media Tags

<a>
Defines a hyperlink.
Attributes:
  • href: Specifies the URL of the link
  • target: Where to open the link (_self, _blank, etc.)
  • rel: Relationship between current document and linked resource (e.g. nofollow, noopener)
  • download: Indicates the link is for downloading
<a href="https://example.com" target="_blank" rel="noopener">Visit Example</a>
Visit Example
<img>
Defines an image. It's a void/self-closing tag (no closing tag).
Attributes:
  • src: URL of the image (required)
  • alt: Alternative text to describe the image (required for accessibility)
  • width, height: Image dimensions
  • title: Tooltip text
  • loading: Lazy loading attribute (lazy, eager)
<img src="https://via.placeholder.com/150" alt="Placeholder Image" width="150" height="150">
<picture>
Container to hold multiple sources for responsive images, falling back to an <img> if none match.
No direct attributes - attributes go on child <source> or <img> tags
<picture>
  <source srcset="small.jpg" media="(max-width: 600px)">
  <source srcset="large.jpg" media="(min-width: 601px)">
  <img src="fallback.jpg" alt="Example Picture">
</picture>
<!-- Picture element chooses image based on viewport width. Images not shown here. --> Picture element demo: will show fallback.jpg if no media match.
<audio>
Embedding audio content. Supports controls, multiple sources, and fallback.
  • controls: Display playback controls
  • autoplay: Play audio automatically on load
  • loop: Repeat audio indefinitely
  • muted: Mute audio initially
  • preload: Hint about loading behavior (auto, metadata, none)
<audio controls>
  <source src="example-audio.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>
<video>
Embedding video content with controls and multiple sources.
  • controls: Display playback controls
  • autoplay: Play video automatically on load
  • loop: Repeat video indefinitely
  • muted: Mute video initially
  • width, height: Dimensions
  • poster: Image shown before playback
  • preload: Hint about loading (auto, metadata, none)
<video width="320" height="240" controls>
  <source src="example-video.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>
<source>
Specifies multiple media resources for <video> and <audio> elements or different image sources in <picture>.
  • src: Path to media file
  • type: Media MIME type (e.g., audio/mpeg, video/mp4)
  • media: Media query (for <picture>)
  • sizes: Size of image (for <picture> and responsive images)
<source src="audio.mp3" type="audio/mpeg">
Used inside media elements; no standalone visual output.

Form and Input Tags

<form>
Defines a form for user input, can contain various input elements.
  • action: URL to which the form data is sent on submission.
  • method: HTTP method for submission (get, post)
  • enctype: Encoding type for form data (e.g. multipart/form-data for file uploads)
  • novalidate: Disable validation
  • autocomplete: Enable or disable autofill
<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>
  <button type="submit">Send</button>
</form>



<input>
Defines a variety of input controls depending on the type attribute. Self-closing tag (void).
  • type: Type of input (e.g., text, password, checkbox, radio, submit, email, number, date, etc.)
  • name: Name of the input control (used when submitting)
  • value: Value of the input
  • placeholder: Hint text displayed inside empty input
  • required: Marks input as required
  • disabled: Disables the input
  • checked: For checkbox/radio to specify checked state
  • min, max, step: For number/date inputs
  • pattern: Regex pattern for validation
<input type="email" name="email" placeholder="Enter your email" required>
<textarea>
Defines a multi-line text input control.
  • rows: Number of visible text lines
  • cols: Visible character width
  • placeholder: Hint text
  • maxlength: Maximum allowed characters
  • required: Mark as required
<textarea rows="4" cols="30" placeholder="Enter comments..."></textarea>
<button>
Defines a clickable button. Can specify its behavior with the type attribute.
  • type: submit (default), button (general button), or reset
  • disabled: Disables the button
<button type="submit">Submit</button>
<button type="button">Click me</button>


<label>
Defines a label for an input element, improves accessibility and clicking label focuses input.
  • for: Id of the input this label is associated with.
<label for="username">Username:</label>
<input id="username" type="text">

Defines an option within a <select> element.
  • value: The value submitted if selected
  • selected: Marks option as initially selected
  • disabled: Disables the option
<optgroup>
Groups related options inside a <select> menu with a label.
  • label: Label for the group
  • disabled: Disables whole group

Represents the result of a calculation or user action.
  • for: IDs of elements that the output depends on

  • form: The form element the output is associated with

Embedded Content Tags

<iframe>
Embeds another HTML page inside the current one.
  • src: URL of the page to embed
  • width, height: Dimensions
  • title: Description for accessibility
  • loading: Lazy loading behavior
  • sandbox: Security restrictions
<iframe src="https://example.com" width="300" height="200" title="Example iframe"></iframe>
<embed>
Embeds external content like PDFs, Flash, or other plug-in content. Void element.
  • src: URL of resource
  • type: MIME type
  • width, height: Dimensions
<object>
Defines a container for embedding external resources (e.g. webpages, PDFs, multimedia). Can fall back to alternative content.
  • data: URL of object
  • type: MIME type
  • width, height: Dimensions
  • form: Associates with form
<param>
Defines parameters for <object> elements.
  • name: Parameter name
  • value: Parameter value
<canvas>
Used as a container to draw graphics on the fly via scripting (usually JavaScript).
  • width, height: Dimensions of the canvas
<canvas id="myCanvas" width="150" height="100" style="border:1px solid #000000;"></canvas>
<script>
var c = document.getElementById("myCanvas");
var ctx = c.getContext("2d");
ctx.fillStyle = "red";
ctx.fillRect(20, 20, 100, 50);
</script>
<svg>
Defines a container for SVG graphics.
  • width, height: Dimensions
  • viewBox: Defines position and dimension in user space
  • xmlns: XML namespace, typically http://www.w3.org/2000/svg
<svg width="100" height="100" xmlns="http://www.w3.org/2000/svg">
  <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
</svg>

Scripting Tags

<script>
Defines client-side script, typically JavaScript.
  • src: URL of external script
  • async: Load script asynchronously
  • defer: Execute script after HTML parsing
  • type: Script type (default is text/javascript)
  • nomodule: Do not execute script in module-supporting browsers
<script>
  console.log('Hello World');
</script>
Console logs "Hello World". No visible output.

Deprecated or Obsolete Tags

<font>
Deprecated. Used to specify font size, color, and face. Use CSS instead.
<center>
Deprecated. Used for centering text. Use CSS (text-align:center) instead.
<big>
Deprecated. Used to make text bigger. Use CSS (font-size) instead.
<strike>
Deprecated. Use <del> or <s> instead. Defines strikethrough text.
<applet>
Deprecated. Use <embed> or <object> instead. Defines an embedded applet.
<basefont>
Deprecated. Use CSS to specify default font styles.
<dir>
Deprecated. Use <ul> instead. Defines a directory list.
<frame>
Deprecated. Use <iframe> or CSS layouts instead. Defines a window in a frameset.
<frameset>
Deprecated. Use CSS layouts instead. Defines a set of frames.
<acronym>
Deprecated in HTML5. Use <abbr> instead.
<tt>
Deprecated. Use CSS to specify monospace font instead.
